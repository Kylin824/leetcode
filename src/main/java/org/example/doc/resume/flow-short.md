# OA 审批协同平台 · 工作流引擎模块

**技术栈**：Java / Spring Boot、Flowable 6.7.1（BPMN 2.0）、Redisson、XXL-JOB、MySQL、Redis、Feign、Kafka

## 项目简介
公司自研 OA 审批平台核心，基于 Flowable 6.7.1 深度定制的工作流引擎。为预算、合同、发票、充值、薪酬等 10+ 业务域提供统一流程编排，沉淀 34 套 BPMN 模板，支撑发起、多级审批、会签、驳回、撤回、转办、加签、终止全生命周期，并以分布式锁、流程热升级、异步解耦保障高并发一致性与稳定性。

## 核心工作与亮点
- **引擎深度定制**：通过自定义事件监听、任务拦截器（离职自动转办/审批人补全）、行为工厂、BPMN 解析器（单节点多人自动升级为并行会签），以及 EL 表达式 `fn` 桥接规则引擎动态算人，实现「流程如何跑」与「谁来审批」解耦，规则变更零改码。
- **高并发审批一致性**：主单级 Redisson 分布式锁统一 key 入口，事务提交后才释放（watchdog 续期、多 key 排序防死锁）；审批 `setNx` 唯一值 + 审计表幂等，杜绝并发重复审批。
- **在途流程热升级**：基于 `ProcessMigrationService` 做节点映射预检 + 迁移，配合 XXL-JOB 批量平滑升级在途单据，无需撤回重提；预检失败明确提示可回退。
- **全生命周期编排**：22 个流程 Service 覆盖提交/审批/会签/驳回/撤回/转办/加签/终止；撤回、转步骤以命令模式按场景分派执行。
- **模块解耦**：引擎只管流程，业务回调经 Feign / 事件 / Kafka 异步触达预算、合同、发票等模块；18 个 XXL-JOB 承接自动审批、流程升级、数据同步等调度。
- **自研流程图渲染**：不启动真实实例，遍历流程定义并解析表达式生成实时状态流程图。

## 可量化成果
34 套 BPMN 流程模板（10+ 业务域）｜22 个流程 Service｜18 个定时任务｜Flowable 6.7.1
