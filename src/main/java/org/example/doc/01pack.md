## 什么是 0-1 背包问题？

**问题描述：**
有一个背包，它的容量为 `W`。现在有 `n` 个物品，每个物品有：
- 重量 `w[i]`
- 价值 `v[i]`

每个物品**要么选（放入背包），要么不选**，不能分割，这就是"0-1"的含义（0表示不选，1表示选）。

问：在不超过背包容量的前提下，如何选择物品使得**总价值最大**？

**举个生活例子：**
假设你有一个承重 5kg 的背包，现在有 3 件物品：
- 物品1：重量 2kg，价值 6元
- 物品2：重量 3kg，价值 10元
- 物品3：重量 4kg，价值 12元

问：怎么装价值最大？

答案：选物品1+物品2 = 重量5kg，价值16元（不选物品3，虽然单价高但会超重）。

---

## 核心思路：动态规划

### 状态定义

定义 `dp[i][j]` 表示：**考虑前 `i` 个物品，背包容量为 `j` 时，能获得的最大价值**。

- `i` 范围：0 ~ n（0表示一个物品都不考虑）
- `j` 范围：0 ~ W（0表示背包容量为0）

### 状态转移方程

对于第 `i` 个物品（下标从1开始），我们面临两个选择：

1. **不选它**：`dp[i][j] = dp[i-1][j]`（继承上一个状态）
2. **选它**（前提是 `j >= w[i]`）：`dp[i][j] = dp[i-1][j-w[i]] + v[i]`（腾出空间再装它）

取两者的最大值：
```
dp[i][j] = max(dp[i-1][j], dp[i-1][j-w[i]] + v[i])   // 当 j >= w[i] 时
dp[i][j] = dp[i-1][j]                                 // 当 j < w[i] 时
```

## Java 实现（二维 DP 版本）

先看最直观的二维数组版本，更容易理解：

```java
public class Knapsack01 {
    public static int knapsack(int W, int[] weights, int[] values) {
        int n = weights.length;
        // dp[i][j] 表示前 i 个物品，容量 j 的最大价值
        int[][] dp = new int[n + 1][W + 1];
        
        // i 从 1 开始，代表第 1 个物品到第 n 个物品
        for (int i = 1; i <= n; i++) {
            for (int j = 0; j <= W; j++) {
                // 当前物品重量 (注意索引偏移，因为 i 从1开始)
                int weight = weights[i - 1];
                int value = values[i - 1];
                
                if (j < weight) {
                    // 装不下，只能不选
                    dp[i][j] = dp[i - 1][j];
                } else {
                    // 装得下，取"不选"和"选"的最大值
                    dp[i][j] = Math.max(dp[i - 1][j], dp[i - 1][j - weight] + value);
                }
            }
        }
        
        // 返回考虑所有物品、容量 W 时的最大价值
        return dp[n][W];
    }
    
    public static void main(String[] args) {
        int W = 5;
        int[] weights = {2, 3, 4};
        int[] values = {6, 10, 12};
        System.out.println(knapsack(W, weights, values)); // 输出 16
    }
}
```

## 空间优化：一维 DP

观察二维 DP 的递推关系：
- `dp[i][j]` 只依赖于 `dp[i-1][...]`，即**上一行**的数据
- 因此我们可以用**一维数组**滚动更新，节省空间

**关键点：** 内层循环必须**从大到小（倒序）** 遍历！

**为什么必须倒序？**
- 因为一维数组中，`dp[j]` 的更新依赖于 `dp[j - weight]`
- 如果正序遍历，`dp[j - weight]` 可能已经在**当前物品**的同一轮循环中被更新过了，导致**同一个物品被多次使用**（变成完全背包问题）
- 倒序遍历可以保证 `dp[j - weight]` 仍然是**上一个物品**的状态

### 一维 DP 代码（更常用）

```java
public class Knapsack01 {
    public static int knapsack(int W, int[] weights, int[] values) {
        int n = weights.length;
        int[] dp = new int[W + 1];
        
        for (int i = 0; i < n; i++) {
            // 倒序遍历！从 W 到 weights[i]，确保每个物品只用一次
            for (int j = W; j >= weights[i]; j--) {
                dp[j] = Math.max(dp[j], dp[j - weights[i]] + values[i]);
            }
        }
        
        return dp[W];
    }
    
    public static void main(String[] args) {
        int W = 5;
        int[] weights = {2, 3, 4};
        int[] values = {6, 10, 12};
        System.out.println(knapsack(W, weights, values)); // 输出 16
    }
}
```

## 执行过程示例

用上面的例子（W=5，物品：2/6、3/10、4/12）走一遍一维DP：

**初始状态：** `dp = [0, 0, 0, 0, 0, 0]`

**第1个物品（重量2，价值6）：** 倒序 j=5→2
- j=5: dp[5]=max(0, dp[3]+6=0)=0
- j=4: dp[4]=max(0, dp[2]+6=0)=0
- j=3: dp[3]=max(0, dp[1]+6=0)=0
- j=2: dp[2]=max(0, dp[0]+6=6)=6
  结果：dp = [0, 0, 6, 0, 0, 0]

**第2个物品（重量3，价值10）：** 倒序 j=5→3
- j=5: dp[5]=max(0, dp[2]+10=16)=16
- j=4: dp[4]=max(0, dp[1]+10=0)=0
- j=3: dp[3]=max(0, dp[0]+10=10)=10
  结果：dp = [0, 0, 6, 10, 0, 16]

**第3个物品（重量4，价值12）：** 倒序 j=5→4
- j=5: dp[5]=max(16, dp[1]+12=0)=16
- j=4: dp[4]=max(0, dp[0]+12=12)=12
  结果：dp = [0, 0, 6, 10, 12, 16]

最终 `dp[5] = 16` ✅

## 0-1背包的变体与应用

| 变体类型 | 描述 | 示例 LeetCode |
|---------|------|---------------|
| **恰好装满** | 问能否恰好装满背包 | LeetCode 416（分割等和子集） |
| **求方案数** | 问有多少种方式装满 | LeetCode 494（目标和） |
| **最小/最大价值** | 常规0-1背包 | 经典问题 |
| **二维费用** | 背包有重量和体积两个限制 | 需要三维DP或二维滚动 |

---

## 常见易错点总结

| 易错点 | 正确做法 |
|--------|---------|
| 遍历顺序 | **一维DP必须倒序**，二维正序倒序都可以 |
| 初始化 | 求最大价值时初始化为0；求"恰好装满"时需初始化为 -∞ |
| 边界条件 | 内层循环 `j >= weights[i]`，容量不够时跳过 |
| 物品索引 | 一维DP直接 `weights[i]`，二维时注意 `i` 从1开始 |

---

## 与完全背包的对比

| 特性 | 0-1背包 | 完全背包 |
|------|---------|----------|
| 物品数量 | 每个物品只能用1次 | 每个物品可以用无限次 |
| 内层遍历 | **倒序**（从大到小） | **正序**（从小到大） |
| 典型题目 | LeetCode 416、494 | LeetCode 322、518 |

## 举一反三：如果你需要输出具体选了哪些物品

如果不仅要最大价值，还要知道选了哪些物品，可以用二维DP记录路径，然后倒推：

```java
// 伪代码
int[][] dp = new int[n+1][W+1];
// ... 正常填充 dp

// 倒推选中的物品
List<Integer> selected = new ArrayList<>();
int i = n, j = W;
while (i > 0 && j > 0) {
    if (dp[i][j] != dp[i-1][j]) {
        selected.add(i-1);  // 选中了第 i 个物品
        j -= weights[i-1];
    }
    i--;
}
```
